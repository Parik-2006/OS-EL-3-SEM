üéØ Project Objective

In High-Performance Computing (HPC) and Cloud systems, standard CPU schedulers fail to handle mixed workloads efficiently. This project introduces Weighted Dynamic Scoring (WDS), a hybrid algorithm designed to balance three competing goals simultaneously:

  • Speed: Minimizing Turnaround Time (like SJF).
  • Fairness: Preventing Starvation (like FIFO/FCFS).
  • Urgency: Respecting User Priority (like Priority Scheduling).

Instead of a rigid rule (like "shortest job first"), WDS calculates a real-time score for every waiting process. The process with the highest score wins the CPU.

Wwait (Aging) Ensures that if a task waits for a long time, its score grows massive, forcing the CPU to pick it. (Solves Starvation).

Wprio (Urgency) Allows critical tasks (e.g., "System Crash Alert") to override the math and run immediately.

‚ùå The Problem

FCFS: Causes "Convoy Effect" (UI freezes behind background tasks).

SJF: Causes "Starvation" (Long videos never process if short texts keep coming).

‚úÖ The WDS Solution

Hybrid Approach: Acts like SJF when the system is busy to clear traffic, but switches to Fairness mode for old tasks.

Cooperative: Uses "Yield Points" to allow high-priority interruptions.
