üìñ Project Objective

In High-Performance Computing (HPC) and Cloud systems, standard CPU schedulers fail to handle mixed workloads efficiently. This project introduces Weighted Dynamic Scoring (WDS), a hybrid algorithm designed to balance three competing goals simultaneously:

  • Speed: Minimizing Turnaround Time (like SJF).
  • Fairness: Preventing Starvation (like FIFO).
  • Urgency: Respecting User Priority (like Priority Scheduling).

Wait Weight (Aging) Ensures that if a task waits for a long time, its score grows massive, forcing the CPU to pick it. (Solves Starvation).
Burst Weight (Speed) Gives a bonus to short jobs. This clears the queue quickly and lowers average system latency.
Priority Weight (Urgency) Allows critical tasks (e.g., "System Crash Alert") to override the math and run immediately.
